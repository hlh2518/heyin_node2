## TVBox JS规则 同步 → 异步 改造要点

### 一、函数定义方式改变
```javascript
// 同步写法
一级: `js: ...复杂JS代码...`

// 异步写法  
一级: async function (tid, pg, filter, extend) {
    // 异步代码
}
```

### 二、HTTP请求方式改变
```javascript
// 同步写法（在JS字符串中）
let html = request(input);

// 异步写法（在async函数中）
let html = await request(input, {headers: this.headers});
```

### 三、数据获取和处理
```javascript
// 同步写法 - 直接使用VODS全局变量
VODS = videos;

// 异步写法 - 直接return数组
return videos;
```

### 四、lazy函数的特殊处理
```javascript
// 同步写法 - 字符串形式
lazy: `js: ...播放解析代码...`

// 异步写法 - 异步函数形式  
lazy: async function (flag, id, flags) {
    // 异步播放解析
    let html = await request(url, {headers: this.headers});
    return {parse: 0, url: playUrl, jx: 0};
}
```

### 五、重要注意事项

#### 1. **所有TVBox回调函数都需要改**
- `推荐`、`一级`、`二级`、`搜索`、`lazy` 都需要改为 `async function`

#### 2. **正确传递headers**
```javascript
// 必须带上this.headers
await request(url, {headers: this.headers});
```

#### 3. **返回值格式要正确**
- 一级/推荐/搜索：返回 `[{vod_id, vod_name, vod_pic, vod_remarks}]`
- 二级：返回完整的 `VOD` 对象
- lazy：返回 `{parse: 0, url: playUrl, jx: 0}`

#### 4. **错误处理很重要**
```javascript
try {
    // 异步操作
    let html = await request(url);
    let data = JSON.parse(html);
    return data;
} catch (e) {
    console.log("错误:", e);
    return []; // 或 return {}
}
```

#### 5. **注意变量作用域**
- 避免在异步函数中使用全局变量
- 通过 `this` 访问规则配置
- 使用闭包或局部变量

### 六、改造步骤清单

1. **✅ 函数签名改造**
   - 所有函数改为 `async function`
   - 移除JS字符串包装

2. **✅ 请求改造**
   - `request()` 前加 `await`
   - 添加headers参数

3. **✅ 返回值改造**
   - 去掉 `VODS = ` 赋值
   - 直接 `return` 数据

4. **✅ lazy函数改造**
   - 改为异步函数
   - 正确处理播放地址获取

5. **✅ 错误处理添加**
   - 添加try-catch
   - 返回默认值

6. **✅ 辅助函数检查**
   - 确保辅助函数在作用域内
   - 处理可能的undefined

### 七、常见问题解决

#### Q1: 一级目录显示"加载失败"
- 检查函数是否返回数组格式
- 检查异步请求是否成功
- 查看控制台错误信息

#### Q2: 二级目录返回空
- 检查二级函数是否正确解析数据
- 确保返回完整的VOD对象
- 检查detailUrl是否正确构建

#### Q3: 播放报错
- lazy函数必须改为异步
- 检查播放地址获取逻辑
- 添加详细的调试日志

#### Q4: 分类筛选失效
- 确保filter参数正确传递
- 检查filter_url模板
- 验证API参数格式

console.log("请求URL:", url);

2. **检查控制台输出**
- 查看请求日志

3. **逐步测试**
- 先测试推荐/一级
- 再测试二级
- 最后测试播放

### 九、性能优化建议

1. **合理使用缓存**
```javascript
// 可以缓存一些不常变的数据
let cacheKey = 'bili_home';
let cached = localStorage.getItem(cacheKey);
if (cached) return JSON.parse(cached);
```

2. **批量请求处理**
```javascript
// 使用Promise.all并行请求
let promises = urls.map(url => request(url));
let results = await Promise.all(promises);
```

3. **错误重试机制**
```javascript
async function requestWithRetry(url, retries = 3) {
    for (let i = 0; i < retries; i++) {
        try {
            return await request(url);
        } catch (e) {
            if (i === retries - 1) throw e;
            await sleep(1000);
        }
    }
}
```

### 总结
同步改异步的核心是：
1. **函数定义** → 改为 `async function`
2. **HTTP请求** → 改为 `await request()`
3. **数据返回** → 改为 `return` 而不是赋值全局变量
4. **错误处理** → 必须添加try-catch
5. **lazy特殊** → 必须也改为异步

只要遵循这个模式，大部分TVBox JS规则都可以顺利从同步迁移到异步框架。
